# Contributing to the Epistemic Stewardship Framework

Thank you for your interest in contributing. The ESF is an open framework for maintaining intellectual ownership in AI-assisted academic work, and community input strengthens it.

## How to Contribute

### Report Issues

Found a broken link, unclear instruction, or factual error? [Open an issue](https://github.com/nmadrid27/Epistemic-Stewardship-Framework-ESF-/issues) with:

- A clear description of the problem
- The file and section where you found it
- A suggested fix, if you have one

### Suggest Improvements

Have an idea for a new template, a better workflow step, or an adaptation for your discipline? Open an issue tagged `enhancement` and describe:

- What you want to change or add
- Why the current version falls short
- How your suggestion serves the framework's core purpose (intellectual ownership in AI-assisted work)

### Submit Changes

1. Fork the repository
2. Create a branch (`git checkout -b your-change`)
3. Make your edits
4. Submit a pull request with a clear description of what changed and why

### Adapt for Your Institution

The ESF is designed to be institution-agnostic. If you adapt the framework for your context, consider sharing your adaptation:

- Open a PR adding your adaptation to a `community/` folder
- Or link to your fork in a GitHub Discussion

## Guidelines

- **Keep it concrete.** The ESF values specificity over abstraction. Contributions should follow the same principle.
- **Preserve the two-level architecture.** Level 1 (Content Production) and Level 2 (Epistemic Development) serve different audiences. Changes should respect this distinction.
- **Cite sources.** If your contribution references research, include full citations. The ESF has zero tolerance for fabricated references.
- **Write clearly.** Plain language, active voice, short sentences. See the framework documents for the register.
